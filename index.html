Why WebAssembly?

  • Completely sandboxed per-library filesystem: never worry about dependencies reading your secret keys
  • Small and lightweight: Containers > 1 MB large for deployment on mobile platforms
  • Runtime-agnostic packaging and ABI: Using Java, but need a C# library? Just import it with WebAssembly!
  • Single binary artifact: no more complicated container-internal network configuration
  • WASM runs on all devices including the browser, independent of platform support
  • Integrated package management for cross-language packages
